# My Cat V0.4 桌面环境感知互动产品方案

## 1. 版本定位

V0.4 的目标不是继续增加“桌宠功能数量”，而是让用户明显感觉到：

> 这只小猫不是浮在屏幕上的动画，而是真的生活在我的桌面环境里。

这一版围绕四类桌面对象做互动：

- 鼠标：看见用户、短暂跟随视线、被点击后回应更明确。
- 窗口：知道窗口靠近、移动、遮挡，会醒来、让开、停留或探头。
- 任务栏：把底部任务栏当成可停留、散步、趴着的桌面边界。
- 拖动：用户移动小猫时，不再像拖图片，而像把小猫抱起又放下。

## 2. 产品原则

### 2.1 保持安静陪伴

所有新互动都应当短、轻、可打断，不能让小猫变成一直抢注意力的桌面玩具。

### 2.2 先做“看得出来”的体验

优先实现用户一眼能感知的行为：

- 点击后看向鼠标。
- 拖动时有抱起和放下。
- 窗口移动靠近时小猫醒来让开。
- 小猫偶尔在任务栏边缘坐着或趴着。

### 2.3 避免过早追求真实遮挡

“躲在窗口后只露脑袋”可以用两种方案：

- 方案 A：专用探头美术资产模拟，只显示头部或半身。
- 方案 B：真实窗口层级和裁剪遮挡。

V0.4 推荐先采用方案 A。它更稳定、更可控，也更容易保证可爱和不干扰桌面操作。真实遮挡可以作为后续技术验证。

## 3. 目标用户体验

### 3.1 鼠标互动

用户点击小猫时，小猫先播放摸摸回应，然后进入 2 到 4 秒的视线关注状态。此时如果用户轻微移动鼠标，小猫会看向鼠标方向。

用户只是把鼠标移到小猫附近时，小猫可以偶尔抬头看一下，但需要冷却时间，避免用户正常操作桌面时频繁触发。

推荐默认：

- 点击触发：总是允许。
- 鼠标靠近触发：仅在小猫处于坐着状态时允许。
- 视线跟随时长：3 秒。
- 冷却时间：18 秒，沿用当前 V0.3 的鼠标注意节奏。
- 安静模式：保留点击回应，关闭主动鼠标注意。

### 3.2 窗口互动

当小猫在某个窗口边缘附近停留时，如果用户移动或缩放该窗口，小猫会表现出“被惊醒”或“发现窗口动了”，然后走开一小段距离。

小猫也可以偶尔走到前台窗口边缘，坐一下、趴一下、探头看一眼。

推荐默认：

- 只感知当前前台窗口，避免扫描所有窗口造成复杂性和不稳定。
- 只在小猫距离窗口边缘较近时触发让开。
- 让开后进入短冷却，避免用户拖窗口时反复触发。
- 窗口探头先使用专用 PNG 动作资产模拟，不做真实遮挡。

### 3.3 任务栏互动

小猫可以把底部任务栏上方当作一条“桌面边缘”：

- 偶尔沿任务栏上方慢慢走。
- 偶尔在任务栏上方坐着。
- 偶尔趴在任务栏上方休息。

这一行为必须克制。任务栏是用户高频操作区域，小猫不能长期挡住开始菜单、应用图标、系统托盘和时间区域。

推荐默认：

- 初版只支持底部任务栏。
- 每 5 到 8 分钟最多触发一次任务栏兴趣。
- 单次停留不超过 8 秒。
- 安静模式下不主动去任务栏。
- 如果鼠标靠近任务栏区域，小猫应优先让开或结束停留。

### 3.4 抱起与放下

当前拖动已经能记录放置点。V0.4 要把它升级为一个完整感受：

1. 用户按住小猫并移动超过轻微阈值。
2. 小猫播放“被抱起”动作。
3. 拖动过程中保持“被抱着”的姿态。
4. 松手后播放“落地/安顿”动作。
5. 显示轻反馈，例如“就待这儿”。

推荐默认：

- 按下后不立刻进入抱起，移动超过 6 到 10 像素再触发，避免普通点击误判。
- 抱起动作 300 到 600 毫秒。
- 拖动中使用独立 held 姿态，不循环复杂动画。
- 放下后继续沿用 3 分钟放置点记忆。

## 4. 功能范围

### 4.1 V0.4 必做

| 模块 | 功能 | 验收标准 |
|---|---|---|
| 鼠标视线 | 点击后短暂看向鼠标 | 点击后 2 到 4 秒内有明确视线关注 |
| 拖动过程 | 抱起、拖动中、放下 | 拖动不再像移动图片，有明确过程感 |
| 窗口让开 | 前台窗口移动靠近时醒来走开 | 小猫在窗口边缘附近时能在 1 秒内响应 |
| 任务栏停留 | 偶尔在底部任务栏上方走、坐或趴 | 不长期遮挡任务栏，安静模式下不主动触发 |
| 美术 manifest | 新动作纳入严格校验 | 缺失资源启动时报错，不静默回退 |
| 测试记录 | 增加互动指标计数 | 能看到点击视线、拖动、窗口让开、任务栏停留次数 |

### 4.2 V0.4 可选

| 功能 | 建议 |
|---|---|
| 窗口后探头 | 若美术资产能及时产出，则作为 V0.4 Plus 接入 |
| 多方向视线 | 先做左、右、上、正面四类；后续再细分 |
| 多屏任务栏 | 暂缓，只支持主屏幕 |
| 真实窗口遮挡 | 暂缓，先做技术 spike，不进入默认版本 |

### 4.3 明确不做

- 不做多猫。
- 不做云同步。
- 不做照片生成。
- 不做复杂物理碰撞。
- 不做全局窗口列表的持续重度扫描。
- 不做会长期挡住用户操作的任务栏驻留。

## 5. 新增美术资产方案

V0.4 可以新增资产，不必采用裁剪方案。推荐继续沿用当前 `my-cat` 美术包规范：

- PNG
- RGBA
- 透明背景
- 512 x 512
- 每个动作 16 帧
- 通过 `manifest.json` 声明动作 ID、帧路径、帧时长和循环方式

### 5.1 必需动作

| 动作 ID | 用途 | 说明 |
|---|---|---|
| `mouse_track` | 视线跟随 | 点击后看向鼠标，可先做通用版本 |
| `drag_lift` | 抱起 | 从正常姿态过渡到被抱起 |
| `drag_hold` | 拖动中 | 被用户拖动时保持的姿态 |
| `drag_drop` | 放下 | 松手落地和安顿 |
| `window_startle` | 窗口惊动 | 窗口移动靠近时醒来 |
| `window_avoid` | 走开 | 从窗口边缘离开 |
| `taskbar_sit` | 任务栏坐着 | 在任务栏上方短暂停留 |
| `taskbar_lie` | 任务栏趴着 | 在任务栏上方短暂休息 |

### 5.2 推荐动作

| 动作 ID | 用途 | 说明 |
|---|---|---|
| `mouse_track_left` | 看向左侧鼠标 | 如果美术能支持方向差异 |
| `mouse_track_right` | 看向右侧鼠标 | 如果美术能支持方向差异 |
| `mouse_track_up` | 看向上方鼠标 | 鼠标在小猫上方时更自然 |
| `peek_left` | 从窗口左边探头 | 用专用资产模拟窗口遮挡 |
| `peek_right` | 从窗口右边探头 | 用专用资产模拟窗口遮挡 |
| `peek_top` | 从窗口上方探头 | 适合趴在窗口边缘 |
| `window_perch` | 窗口边缘停留 | 替代或增强当前 `window_linger` |

### 5.3 美术表现建议

- 抱起动作要偏柔和，不要显得小猫被粗暴拎起。
- 拖动中姿态要稳定，避免用户拖动时动画抖动。
- 探头动作建议只显示头部、前爪或半个身体，让用户一眼理解“它躲在后面”。
- 任务栏动作应尽量压低重心，减少对任务栏图标的遮挡。
- 鼠标视线动作可以先通过头部方向、眼睛方向和耳朵轻微变化表达，不必全身大动作。

## 6. 工程方案

### 6.1 新增桌面环境感知层

建议新增 `DesktopEnvironmentService`，统一提供：

- 当前鼠标屏幕坐标。
- 主屏幕工作区和完整屏幕区域。
- 底部任务栏矩形。
- 当前前台窗口矩形。
- 前台窗口是否发生移动或缩放。

UI 层可以负责读取 Windows API，但行为判断应尽量保持在清晰的服务或控制器中，避免散落在窗口事件里。

### 6.2 行为优先级

建议优先级从高到低为：

1. 用户拖动。
2. 点击回应和视线跟随。
3. 真实猫观察反馈。
4. 窗口移动让开。
5. 任务栏兴趣。
6. 鼠标靠近注意。
7. 自动坐、睡、走。

安静模式下保留用户直接触发的点击、拖动和记录反馈，但关闭或大幅减少窗口兴趣、任务栏兴趣和鼠标靠近注意。

### 6.3 新增状态建议

| 状态 | 说明 |
|---|---|
| `MouseTrack` | 短暂看向鼠标 |
| `DragLift` | 被抱起过渡 |
| `DragHold` | 拖动中 |
| `DragDrop` | 放下过渡 |
| `WindowStartle` | 被窗口惊动 |
| `WindowAvoid` | 从窗口边缘走开 |
| `TaskbarVisit` | 任务栏边缘活动 |
| `WindowPeek` | 从窗口后探头，可选 |

### 6.4 指标与本地数据

建议在现有 `interaction-metrics.json` 中新增：

- `mouseTrackCount`
- `dragLiftCount`
- `dragDropCount`
- `windowAvoidCount`
- `taskbarVisitCount`
- `windowPeekCount`

这些指标不做复杂报表，只用于开发和试玩判断新体验是否真的被触发。

## 7. 验收标准

### 7.1 鼠标互动验收

- 点击小猫后，小猫能明显看向鼠标方向。
- 视线跟随结束后能回到原本状态。
- 鼠标频繁经过时不会一直打断小猫。

### 7.2 拖动验收

- 轻点小猫不会误触发拖动。
- 拖动时能看到抱起和拖动中姿态。
- 松手后能看到放下或安顿。
- 放置点记忆仍然有效。

### 7.3 窗口互动验收

- 小猫在窗口边缘附近时，移动窗口能触发醒来或让开。
- 窗口不在小猫附近时，不应频繁触发。
- 前台窗口关闭、最小化或句柄失效时程序不崩溃。

### 7.4 任务栏互动验收

- 小猫能偶尔在底部任务栏上方活动。
- 单次停留时间短，不明显妨碍点击任务栏。
- 安静模式下不主动去任务栏。

### 7.5 稳定性验收

- 连续运行 30 分钟无明显崩溃或卡死。
- 快速移动鼠标、拖动窗口、拖动小猫时无明显异常跳变。
- 缺失新增资产时启动校验能给出明确错误。

## 8. 开发拆解

### 阶段 A：抱起放下闭环

1. 新增 `drag_lift`、`drag_hold`、`drag_drop` 动作资产和 manifest。
2. 拖动识别增加移动阈值。
3. 拖动状态拆分为抱起、拖动中、放下。
4. 保持当前放置点记忆和反馈。

### 阶段 B：鼠标视线跟随

1. 新增 `mouse_track` 动作。
2. 点击后进入视线跟随窗口期。
3. 根据鼠标相对位置决定朝向或动作变体。
4. 增加冷却和安静模式规则。

### 阶段 C：窗口移动让开

1. 抽出前台窗口矩形读取逻辑。
2. 记录窗口矩形变化。
3. 判断小猫是否靠近窗口边缘。
4. 触发 `window_startle` 和 `window_avoid`。

### 阶段 D：任务栏活动

1. 计算底部任务栏区域。
2. 新增任务栏目标点选择。
3. 接入 `taskbar_sit`、`taskbar_lie`。
4. 增加停留时长、冷却和鼠标靠近让开。

### 阶段 E：窗口探头

1. 新增 `peek_left`、`peek_right`、`peek_top`。
2. 在窗口边缘选择探头位置。
3. 使用专用 PNG 模拟遮挡。
4. 根据试玩反馈决定是否继续研究真实遮挡。

## 9. 待确认决策

### 9.1 探头方案

推荐默认：专用 PNG 探头资产模拟窗口遮挡。

需要确认：是否接受 V0.4 先不做真实窗口层级遮挡？

### 9.2 任务栏活跃度

推荐默认：低频出现，单次不超过 8 秒。

需要确认：你希望它更安静，还是更明显地在任务栏上活动？

### 9.3 抱起手感

推荐默认：像“轻轻抱起”，不是“拎起来”。

需要确认：美术风格更偏温柔陪伴，还是可以有一点调皮夸张？

### 9.4 美术优先级

推荐默认：先做抱起放下、鼠标视线、窗口让开；探头和任务栏趴着作为第二批。

需要确认：如果美术制作时间有限，是否按这个顺序推进？

## 10. 推荐结论

V0.4 最值得优先押注的是：

1. 抱起放下。
2. 点击后的视线跟随。
3. 窗口移动时醒来让开。
4. 低频任务栏活动。

这四件事都直接服务于“它真的在我的桌面生活”这个感受。窗口后探头很有记忆点，但建议建立在专用美术资产上，先做稳定可爱的模拟版本，再考虑更复杂的真实遮挡。
